常规提交和代码风格规则：

1. 调试和开发：
   - 没有调试语句（console.log、打印调试）
   - 没有注释掉的代码块
   - 代码应该是生产就绪的

2. 代码质量：
   - 遵循整洁代码原则
   - 函数和方法有良好的文档
   - 代码有适当的测试

3. 性能：
   - 没有明显的性能瓶颈
   - 使用高效的算法和数据结构
   - 没有不必要的计算

4. 安全：
   - 没有安全漏洞
   - 适当的输入验证
   - 安全地处理用户数据

请分析代码更改并按照以下 JSON 格式返回结果：
{
    "status": "PASS" 或 "FAIL",
    "issues": [
        {
            "file": "文件名",
            "line": 行号,
            "rule": "违反的规则类别",
            "description": "问题的详细描述"
        }
    ],
    "summary": "整体评估总结"
}
